## 一、项目概述

### 1.1 项目名称
**大树（ArroDashu）** — 树木品种管理与知识库平台

### 1.2 项目背景
随着城市绿化、园林景观、林业种植等行业的快速发展，对树木品种的科学管理需求日益增长。传统的树木品种管理方式存在信息分散、检索困难、数据不完整等问题。本项目旨在构建一个集树木品种信息录入、查询、管理、知识分享于一体的综合平台。

### 1.3 项目目标
- 建立完善的树木品种数据库，涵盖形态特征、生长习性、养护要点等信息
- 提供便捷的信息检索和可视化展示功能
- 支持多维度分类管理和标签体系
- 构建知识分享社区，促进经验交流

### 3.2 核心功能模块

#### 3.2.1 用户模块
- **用户注册/登录**: 支持邮箱、手机号注册，JWT Token认证
- **个人中心**: 头像、昵称、简介、密码修改
- **权限管理**: RBAC角色权限模型（超级管理员、管理员、普通用户、访客）
- **收藏关注**: 收藏感兴趣的品种，关注其他用户

#### 3.2.2 品种管理模块
- **品种录入**: 支持表单录入、批量导入（Excel/JSON）
- **品种查询**: 多条件筛选、模糊搜索、高级搜索
- **品种详情**: 完整信息展示（基本信息、形态特征、生长习性、养护要点）
- **分类管理**: 科、属、种多级分类，自定义标签
- **图片管理**: 多图上传、图片压缩、缩略图生成

#### 3.2.3 知识中心模块
- **百科文章**: 树木品种详细介绍文章
- **养护指南**: 浇水、施肥、修剪、病虫害防治指南
- **病虫害库**: 常见病虫害信息库，关联受影响品种
- **问答社区**: 用户提问、专家解答、经验分享

#### 3.2.4 数据管理模块
- **数据导入**: Excel模板导入、API数据同步
- **数据导出**: Excel、PDF、JSON格式导出
- **数据备份**: 定时自动备份、手动备份
- **开放API**: RESTful API供第三方调用

#### 3.2.5 系统管理模块
- **用户管理**: 用户列表、状态管理、批量操作
- **角色管理**: 角色定义、权限分配
- **日志审计**: 操作日志、登录日志、异常日志
- **系统配置**: 站点配置、邮件配置、存储配置

#### 3.2.6 统计分析模块
- **品种统计**: 按科属统计、按地区统计、按季节统计
- **访问分析**: PV/UV统计、热门品种排行、用户行为分析
- **趋势预测**: 基于历史数据的趋势分析
- **报表导出**: 可视化报表、数据报表导出

## 四、数据库设计

### 4.1 核心数据表

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| `sys_user` | 用户表 | id, username, email, phone, password, avatar, status |
| `sys_role` | 角色表 | id, role_name, role_code, description |
| `sys_user_role` | 用户角色关联表 | user_id, role_id |
| `tree_species` | 品种表 | id, name, latin_name, family_id, genus_id, description |
| `tree_family` | 科表 | id, name, latin_name, description |
| `tree_genus` | 属表 | id, name, latin_name, family_id |
| `tree_image` | 品种图片表 | id, species_id, url, type, sort_order |
| `tree_tag` | 标签表 | id, name, color |
| `tree_species_tag` | 品种标签关联表 | species_id, tag_id |
| `article` | 文章表 | id, title, content, author_id, category, status |
| `user_favorite` | 用户收藏表 | user_id, species_id, create_time |
| `operation_log` | 操作日志表 | id, user_id, operation, method, params, ip |

### 4.2 数据库选型说明

- **MySQL**: 主数据库，存储业务数据，支持事务
- **Redis**: 缓存热点数据、会话存储、限流计数
- **Elasticsearch**: 全文搜索品种名称、描述、文章内容
- **MinIO**: 存储图片、文件等非结构化数据

## 五、接口设计

### 5.1 RESTful API 规范

```
GET    /api/v1/species          # 获取品种列表
GET    /api/v1/species/{id}     # 获取品种详情
POST   /api/v1/species          # 创建品种
PUT    /api/v1/species/{id}     # 更新品种
DELETE /api/v1/species/{id}     # 删除品种
GET    /api/v1/species/search   # 搜索品种
POST   /api/v1/species/import   # 批量导入
GET    /api/v1/species/export   # 批量导出
```

### 5.2 接口安全
- JWT Token 认证
- 接口签名验证
- 请求频率限制（基于Redis）
- SQL注入防护（MyBatis参数化查询）
- XSS攻击防护

## 八、部署方案

### 8.1 开发环境

```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: arrodashu
  redis:
    image: redis:7-alpine
  elasticsearch:
    image: elasticsearch:8.11.0
  rabbitmq:
    image: rabbitmq:3.12-management
  minio:
    image: minio/minio:latest
```

### 8.2 生产环境

- **服务器**: 2核4G 起步，建议 4核8G
- **数据库**: MySQL 8.0 主从架构
- **缓存**: Redis 集群
- **搜索**: Elasticsearch 集群
- **文件存储**: MinIO 分布式部署 或 阿里云OSS
- **负载均衡**: Nginx / 阿里云SLB
- **容器编排**: Docker + Docker Compose / Kubernetes
